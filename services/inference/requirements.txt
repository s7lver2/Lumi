# --extra-index-url below is REQUIRED for torch/torchvision to resolve to a
# CUDA build. Without it, `pip install torch==X.Y.Z` silently grabs PyPI's
# CPU-only wheel (its actual version string is "X.Y.Z+cpu", not "X.Y.Z") —
# confirmed live: RoMa (Laila) verification fell back to CPU despite an
# RTX 4070 SUPER being present, making /verify take ~45s per candidate
# instead of a couple of seconds, reading as a hung request. The exact
# "+cu121" local version pins below force pip to fetch the CUDA build from
# this extra index rather than the ambiguous-but-matching CPU one on PyPI.
#
# torch is pinned to 2.5.1, not 2.3.1 — confirmed live: `pip install` warned
# "romatch 0.1.2 requires torch>=2.5.1, but you have torch 2.3.1", so the
# original 2.3.1 pin was already unsupported by romatch even before the CUDA
# fix, just silently (pip only warns on this, it doesn't refuse to install).
--extra-index-url https://download.pytorch.org/whl/cu121
